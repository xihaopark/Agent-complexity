## Takeaways for E3

1. The V2 lift of `star_deseq2_init` is purely down to the leading-column
   name difference (`""` vs `"gene"`). Worth asking A3 / registry curators
   to normalise the first header to `gene` — with that pre-applied the
   byte-identical tier would fire and we'd stop paying the small
   `0.99` cap penalty.
2. `methylkit_load` in the paper arm *did* produce a semantically
   equivalent RDS but the raw bytes differ because of methylKit
   metadata (treatment vector, `pipeline` string). The `rds_semantic`
   tier catches that cleanly; recommend we keep the R sidecar in the E3
   evaluation path.
3. `methylkit_to_tibble` is the single task where V2 is *stricter* than
   V1. E3 should look at the task spec — it looks like the current script
   mirrors only one sample while the reference aggregates all four.
4. Tolerance defaults (`rtol=1e-3`, `atol=1e-5`) are well-calibrated:
   tightening to `1e-6` reduces these upgrades but uncovers no
   regressions; loosening to `1e-2` starts giving tabular credit to
   runs that had obviously different data and should be avoided.
