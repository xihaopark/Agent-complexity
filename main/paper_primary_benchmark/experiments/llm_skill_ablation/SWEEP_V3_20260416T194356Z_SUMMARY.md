## Key takeaways

- **Paper arm is truncated; treat any full-32 paper comparison as lower bound.** Of 32 paper-arm attempts, 23 were killed by HTTP 402 from OpenRouter; the 9 completed runs cover only RNA-seq + methylation families. Any V2 count that shows `paper=5/32` is an artefact of missing runs, not of skill quality. The honest paper comparison is the 9-task restricted table above.
- **Among the three 32/32 arms, `none` edges out `pipeline` and `llm_plan` under V2.** `none` 19/32 pass (71.9% pass-or-better), `pipeline` 17/32 pass (68.8%), `llm_plan` 17/32 pass (68.8%). Mean V2 overall_score: `none` 0.777, `pipeline` 0.763, `llm_plan` 0.734. Lift between best (`none`) and worst (`llm_plan`) is **0.043 mean_score / 2 passes** ‚Äî well within LLM sampling noise at temp 0.1.
- **Skill differentiation is swamped by workspace shortcuts.** `none` scoring above `pipeline`/`llm_plan` is exactly the Recommendation #1 effect flagged in the V2 sweep summary: as long as each task workspace ships its own `run_*.R` alongside `OBJECTIVE.md`, the agent can reconstruct the recipe without any injected skill; adding a skill mostly just adds prompt bulk that can confuse verbose LLM chains. Future sweeps should hide the shipped R scripts before declaring a winning skill source.
- **On the 9-task paper overlap, paper is the top arm** (5 pass vs `none` 4, mean 0.722 vs 0.628). This preserves the V2 sweep's directional finding that paper-derived skills add signal on DESeq2-style RNA-seq tasks, though at N=9 the signal is not publication-strength.
- **V2 lenient lift over V1 is real but modest.** V2 upgrades V1 verdicts on `none` +3, `paper` +2, `pipeline` +2, `llm_plan` +2 tasks. So the BixBench-style tolerance rescues ~7‚Äì10% of V1-failed tasks per arm ‚Äî useful, not decisive.
- **Methylation remains the hardest family.** methylkit tasks cluster at `fail` or `partial_fail` under every arm under V2 (`methylkit_load`/`methylkit_unite`/`methylkit_filt_norm`), because the RDS sidecar can't recover semantic equality when methylKit's S4 slots don't round-trip through the `as.data.frame` path. Only 1‚Äì2/6 methylation tasks pass per arm.
- **Skill audit is clean on every completed run.** All 32/32 `none`, 32/32 `pipeline`, 32/32 `llm_plan`, and 9/9 completed `paper` runs have the correct `arm`, `injected` flag, and `skill_sha256` (matching the manifest-recomputed expected SHA). The 23 `paper` audit failures are *all* attributable to the crashed rollouts (missing `metadata.json`) ‚Äî no cross-arm leakage.
- **Budget:** actual OpenRouter billing ended at the account's hard credit ceiling mid-paper-arm (HTTP 402 returned from the 10th task onwards). Best-effort estimate from trajectory step counts is ‚âà $7 for the 9+32+32+32 = 105 completed runs; OpenRouter's final balance should be consulted for exact spend. No runs were attempted after the 402 storm ‚Äî we halted per the plan's "halt and report if you hit $18" rule.
