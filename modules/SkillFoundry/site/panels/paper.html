Abstract

Modern scientific ecosystems are rich in procedural knowledge across repositories, APIs, scripts, notebooks, documentation, databases, and papers, yet much of this knowledge remains fragmented across heterogeneous artifacts that agents cannot readily operationalize. This gap between abundant scientific know-how and usable agent capabilities is a key bottleneck for building effective scientific agents.

We present SkillFoundry, a self-evolving framework that converts such resources into validated agent skills — reusable packages that encode task scope, inputs and outputs, execution steps, environment assumptions, provenance, and tests. SkillFoundry organizes a target domain as a domain knowledge tree, mines resources from high-value branches, extracts operational contracts, compiles them into executable skill packages, and then iteratively expands, repairs, merges, or prunes the resulting library through a closed-loop validation process. SkillFoundry produces a substantially novel and internally valid skill library, with 71.1% of mined skills differing from existing skill libraries. We demonstrate that these mined skills improve coding agent performance on five of the six MoSciBench datasets, and substantially boost performance on two challenging genomics tasks.

Framework Overview

Method

SkillFoundry is a tree-guided framework for automatically building and maintaining a domain skill library from heterogeneous scientific resources. Its core idea is to use a domain knowledge tree both as a search prior and as the object being updated — turning open-ended skill collection into a closed-loop acquisition process.

1

Domain Knowledge Tree Construction

Represent the target field as a rooted tree where internal nodes denote domains and subdomains, and leaves denote actionable skill targets. Each node stores linked resources, existing skills, and validation status to distinguish well-covered regions from high-value frontiers.

2

Tree-Guided Resource Mining

Given the current tree, select a focus branch and retrieve relevant resources: repositories, databases, APIs, papers, and notebooks. Resource search prioritizes authoritative artifacts grounded in reliable sources rather than generic web text.

3

Skill Compilation

Extract an operational contract from retrieved artifacts and compile it into a reusable skill package. Each skill records its scope, dependencies, inputs, outputs, provenance, and example usage — both human-readable instructions and machine-readable metadata.

4

Multi-Level Validation

Apply three complementary tests: execution testing (skill runs under declared contract), system testing (validates cluster-infrastructure skills), and synthetic-data testing (checks contract completeness and behavioral stability on controlled inputs).

5

Tree Expansion

Skills that pass validation are inserted as candidate new leaves, expanding coverage in the corresponding branch. The tree grows to reflect verified skill availability across the target domain.

6

Tree Refinement & Loop

Skills that fail repeatedly, duplicate existing functionality, or contribute little marginal value are revised, merged, or pruned. The updated tree then guides the next round of mining, yielding a self-evolving library.

Experiments

We evaluate SkillFoundry from two perspectives: the quality of the skill library it builds, and the downstream utility of those skills on scientific tasks. At the library level, we study validity, executability, and novelty. At the task level, we evaluate on MoSciBench and two challenging genomics workflows.

Skill Library Analysis

SkillFoundry contains 286 skills spanning 27 domains and 254 subdomains, mined from 394 resources. All retained skills pass multi-level validation (execution, synthetic-data, and system testing). 71.1% of mined skills are novel relative to existing libraries such as SkillHub and SkillSMP. The remaining 28.9% are either discarded as redundant or merged with existing skills. Skill extraction is the dominant runtime cost; testing and tree updates add minimal overhead.

MoSciBench Evaluation

We evaluate whether SkillFoundry skills improve agent performance on MoSciBench, a benchmark for end-to-end multimodal scientific discovery with LLM agents covering six datasets across climate science, biomedical engineering, cheminformatics, health psychology, population genomics, and earth science. We compare the same coding agent with and without SkillFoundry skills. Adding skills improves performance on five of the six datasets and leaves one unchanged, while maintaining 100% code execution success throughout. Averaged over all datasets, Repo-Acc rises from 61.19% → 66.73% and Paper-Acc from 43.85% → 53.05%.

Genomics Task Evaluation: Cell Type Annotation

We test whether SkillFoundry skills improve agent performance on a realistic single-cell analysis workflow. Agents equipped with SkillFoundry-generated skills are evaluated on cell type annotation for spatial transcriptomics data, a task requiring preprocessing, normalization, and biological interpretation.
